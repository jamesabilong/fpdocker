# speckit/

This directory holds per-feature specification artifacts generated by the
spec-driven development (speckit) workflow.

## Structure

```
speckit/
└── <branch>/          ← auto-created by /speckit.specify
    ├── spec.md        ← feature specification (requirements, user stories, edge cases)
    ├── plan.md        ← implementation plan (architecture, phases, data model)
    ├── tasks.md       ← atomic task list (generated by /speckit.tasks)
    ├── research.md    ← optional background research
    ├── data-model.md  ← optional data model detail
    ├── quickstart.md  ← optional dev quickstart notes
    └── contracts/     ← optional API / schema contracts
```

## Branch naming conventions

Both formats are recognised by the speckit scripts:

| Format | Example | When to use |
|---|---|---|
| `NNN-short-name` | `001-docker-hardening` | Sequential local features |
| `FP-NNN-short-name` | `FP-42-add-healthcheck` | Jira / GitHub issues prefixed with `FP-` |

## Workflow

```
/speckit.specify   → creates speckit/<branch>/ and spec.md
/speckit.plan      → populates plan.md
/speckit.tasks     → generates tasks.md
/speckit.analyze   → validates consistency across the three artifacts
/speckit.implement → executes tasks with full context
```

> **Do not create subdirectories here manually.**
> Run `/speckit.specify` from your editor to scaffold a new feature branch correctly.
